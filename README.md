# Lightweight ML Engineering Workspace

A minimal, self-hostable platform for managing the ML experimentation lifecycle
**after feature engineering is done**: upload an ML-ready CSV → get a Dataset
Health Report → configure preprocessing → train & compare models →
track experiments → promote a model → call a REST inference endpoint.

Built to be resume/portfolio-friendly: it demonstrates real ML-engineering
system design (service-oriented backend, experiment tracking, model registry,
deployment/rollback) without needing MLflow, Docker, or a cloud account.

## Architecture

```
backend/
  app/
    main.py            FastAPI app, router wiring, CORS
    config.py           paths, DB URL, thresholds
    database.py          SQLAlchemy engine/session
    models.py             ORM models: Project, Dataset, Experiment, Deployment
    schemas.py             Pydantic request/response models
    routers/
      projects.py          CRUD for projects
      datasets.py           CSV upload -> Dataset Health Report
      experiments.py         train / list / compare experiments, /algorithms
      deployments.py          promote / rollback / history
      inference.py             REST prediction endpoint
    services/
      dataset_analysis.py    the Dataset Health Report engine
      preprocessing.py        builds sklearn ColumnTransformer from config
      training.py               algorithm registry + train/evaluate
      model_registry.py          joblib save/load, versioned by experiment
  storage/
    datasets/               uploaded CSVs (versioned per project)
    models/                  trained pipelines (preprocessing+model in one file)
frontend/
  src/
    main.jsx, App.jsx        entry point + routing (React Router)
    layouts/WorkspaceLayout.jsx  persistent sidebar + shared data loading per project
    pages/                    Overview, Datasets, Train, Experiments, Deployments, Inference
    components/                Sidebar, Modal, Toast, EmptyState, StatCard, PageHeader, Icon
    context/ToastContext.jsx   toast notification system
    lib/api.js, lib/format.js  API client + formatting helpers
  A proper Vite + React app (needs `npm install`, see Setup below) — not a
  CDN/single-file hack. Persistent left-nav workspace per project, modals with
  Cancel buttons, toasts instead of full-page reloads.
sample_data/
  customer_churn_classification.csv
  house_price_regression.csv
```

**Design notes / why it's built this way**
- Everything the user needs to reproduce an experiment (dataset version,
  preprocessing config, algorithm, hyperparameters, metrics, model artifact
  path) is stored in `Experiment` rows in the database — that's the whole
  "experiment tracking" story.
- Preprocessing + the trained estimator are saved as a **single sklearn
  `Pipeline`** joblib file, so inference automatically re-applies the exact
  same imputation/encoding/scaling used at training time — no drift.
- SQLite is the default database so the project runs with zero external
  services. Switching to PostgreSQL later is a one-line change
  (`DATABASE_URL` env var) because everything goes through SQLAlchemy.
- Alembic isn't wired up in v1 (kept lightweight on purpose) but the ORM
  models are structured so it can be added later without refactoring.
- Optional boosting libraries (`xgboost`, `lightgbm`, `catboost`) are
  auto-detected: install them and they show up in `/algorithms` automatically.

## Setup (Windows, Python 3.12)

```bat
cd ml-workspace\backend
py -3.12 -m venv venv
venv\Scripts\activate
pip install -r requirements.txt
uvicorn app.main:app --reload --port 8000
```

macOS/Linux:
```bash
cd ml-workspace/backend
python3.12 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
uvicorn app.main:app --reload --port 8000
```

The API is now live at `http://localhost:8000` and interactive docs at
`http://localhost:8000/docs` (auto-generated by FastAPI — great to screenshot
for a resume/portfolio).

### Frontend

```bash
cd ml-workspace/frontend
npm install
npm run dev
```

Open the URL it prints (usually `http://localhost:5173`). It talks to the
API at `http://localhost:8000` — change the `API_BASE` constant in
`src/lib/api.js` if your backend runs elsewhere.

The app is a proper multi-page workspace: a persistent sidebar per project
(Overview / Datasets / Train / Experiments / Deployments / Inference), a
breadcrumb showing exactly where you are, modals with explicit Cancel
buttons for creating projects and uploading datasets, and toast
notifications instead of full-page reloads. Routing is client-side
(React Router), so navigating between sections doesn't refetch everything —
only the page you land on updates.

## Deployment

Local dev (`uvicorn --reload` / `npm run dev`) is covered above. For running
this in Docker, or actually deploying it to a public URL (Render, with
Vercel/Netlify as an alternative for the frontend), see **[DEPLOYMENT.md](./DEPLOYMENT.md)**
— it also covers the environment variables (`DATABASE_URL`, `CORS_ORIGINS`,
`VITE_API_BASE`) you need to set for a production build.

## Try it end-to-end

1. Start the backend, open the frontend.
2. Create a project (e.g. "Customer Churn").
3. Upload `sample_data/customer_churn_classification.csv` — you'll get a
   Dataset Health Report flagging missing values, the categorical
   `plan_type` column, and class imbalance.
4. Configure preprocessing (mean imputation, one-hot encoding, standard
   scaling), pick an algorithm (e.g. `random_forest`), and launch training.
5. Train a couple more algorithms (`logistic_regression`, `gradient_boosting`)
   to populate the Experiment Dashboard and compare them.
6. Promote your best experiment to production from the dashboard.
7. Use the Inference Tester to POST a JSON record and get a live prediction
   from the deployed model.
8. Try `sample_data/house_price_regression.csv` in a second project to see
   the regression path (MAE/RMSE/R² instead of accuracy/F1).

## API quick reference

| Method | Path | Purpose |
|---|---|---|
| POST | `/projects` | create a project |
| GET | `/projects` | list projects |
| POST | `/projects/{id}/datasets` | upload CSV -> health report |
| PATCH | `/projects/{id}/datasets/{dataset_id}` | rename a dataset |
| DELETE | `/projects/{id}/datasets/{dataset_id}` | delete a dataset (blocked if experiments depend on it) |
| GET | `/algorithms` | list available algorithms per problem type |
| POST | `/projects/{id}/train` | launch a training run -> Experiment |
| GET | `/projects/{id}/experiments` | list/compare experiments |
| POST | `/projects/{id}/deployments` | promote an experiment to production |
| POST | `/projects/{id}/deployments/rollback/{deployment_id}` | rollback |
| POST | `/projects/{id}/predict` | run inference with the active model |

Full interactive schema: `/docs`.

## Explicitly out of scope (by design)

Raw data ingestion, domain-specific feature engineering, image/NLP/geospatial
preprocessing, distributed training, job queues, cloud storage, and
production monitoring. The system assumes an ML-ready CSV as input and is a
foundation you could extend toward those things later — not a reproduction
of MLflow or an AutoML platform.
